Desafio 1: Teste de Integração de Autenticação

Objetivo: Verificar se a API de Autenticação de Usuários está funcionando corretamente.

Descrição: Crie um teste de integração para garantir que o processo de autenticação de usuários esteja funcionando conforme o esperado. O teste deve incluir os seguintes passos:

Registrar um novo usuário com credenciais válidas.
Fazer login com as credenciais fornecidas durante o registro.
Verificar se o status de resposta é 200 (OK) e se um token de autenticação é retornado.
Usar o token de autenticação para fazer uma solicitação à rota protegida "/profile".
Verificar se o status de resposta é 200 (OK) e se os detalhes do usuário são retornados corretamente.

Desafio 2: Teste de Integração de Criação de Recurso

Objetivo: Garantir que a funcionalidade de criação de recursos esteja correta nas APIs.

Descrição: Crie um teste de integração para verificar se a criação de recursos (como posts, produtos ou pedidos) está funcionando conforme o esperado. O teste deve incluir os seguintes passos:

Enviar uma solicitação POST para a rota responsável pela criação do recurso, fornecendo os dados necessários.
Verificar se o status de resposta é 201 (Created) e se os dados do recurso criado estão corretos na resposta.
Realizar uma solicitação GET para a rota responsável por listar todos os recursos e verificar se o recurso criado está presente na lista.

Desafio 3: Teste de Integração de Atualização de Recurso

Objetivo: Verificar se a funcionalidade de atualização de recursos está funcionando corretamente nas APIs.

Descrição: Crie um teste de integração para garantir que a atualização de recursos esteja funcionando conforme o esperado. O teste deve incluir os seguintes passos:

Realizar uma solicitação GET para obter um recurso específico existente.
Verificar se o status de resposta é 200 (OK) e se os detalhes do recurso são retornados corretamente.
Enviar uma solicitação PUT para a rota responsável pela atualização do recurso, fornecendo os dados atualizados.
Verificar se o status de resposta é 200 (OK) e se os dados do recurso foram atualizados corretamente na resposta.
Realizar uma solicitação GET novamente para verificar se as alterações foram aplicadas corretamente ao recurso.
